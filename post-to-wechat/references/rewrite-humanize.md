# 改写并去除中文草稿的 AI 味

在 URL/X 转微信公众号文章流程中，翻译完成后、生成插图前，使用此参考。

## 触发条件

当用户提出以下需求时，也使用该流程：

- "帮我洗稿这篇文章"
- "改写成原创"
- "降低查重率"
- "去掉 AI 味"

## 标准流程

1. **获取源文本**：使用抓取到的来源、翻译草稿或下载的文章正文。
2. **分析结构**：识别文章类型、叙述人称、核心观点、证据、段落层级、媒体素材和可能的读者意图。
3. **深度改写**：为自然中文、原创性和微信可读性重组并改写。
4. **添加 frontmatter**：已知时包含 `title`、`summary`、`author`、`sourceUrl` 和 `coverImage`。
5. **向后传递**：将改写后的 markdown 发送到插图生成和发布流程。

## 改写策略

### A. 结构重组

- 如果能改善叙事流动，就调整段落顺序。
- 将长段落拆分为更短小节，或合并碎片化观点。
- 在有帮助时转换叙述角度：时间线、问题切入、对比切入或故事切入。
- 保留核心证据，同时改变顺序和解释路径。
- 保留来源叙述人称。如果来源是一手经验，改写后的文章也保持第一人称（`我` / `我们`）。除非来源本身是第三人称报道，否则不要改写成“作者/他/她/某人认为”。
- 不要使用旁观者式解读。避免“作者认为”“这篇文章指出”“他在强调”“原文想表达”“这其实是在说”等措辞。用文章自身的声音，把内容改写成直接经验、直接结论或实用指导。

### B. 语言改写

- 用具体事实替换夸大的表达。
- 删除含糊的权威表述，除非明确给出来源。
- 用具体动作替换含糊动词。
- 去掉广告腔和过度打磨的销售话术。
- 当普通表达足够时，避免高频 AI 词：赋能、闭环、生态、抓手、底层逻辑、范式、沉淀、势能。
- 删除填充短语：事实上、值得注意的是、总体来说、不难发现。
- 用真实结论或行动项替换空洞结尾。
- 避免机械同义词替换；稳定概念使用稳定术语。
- 减少不必要的粗体和模板化列表。

### C. 标题改写

生成 3-5 个覆盖有效角度的备选标题：

- 问题切入：用真实问题制造好奇。
- 数字切入：用数字提升具体度。
- 悬念切入：制造信息差，但不标题党。
- 痛点切入：点出读者的具体问题。
- 禁止使用“不是……而是……”句式做标题，也不要用它的轻微变体（如“不是 X，是 Y”“与其说 X，不如说 Y”）做标题。

选择一个标题写入 frontmatter，并将备选标题保留在 `rewrite-notes.md` 中。

### D. 开头改写

使用以下模式之一改写开头：

- 故事切入：从一个小场景或真实用例开始。
- 数据切入：从一个有冲击力的数字或具体观察开始。
- 痛点切入：直接点出读者可能遇到的问题。
- 问题切入：用一个反直觉问题开场。

### E. 可选 SEO

- 在标题和第一节中自然放入核心关键词。
- 只有在不损害可读性时，才在标题中使用长尾关键词。
- 关键词密度保持自然；不要堆砌关键词。

## AI 痕迹检查清单

生成插图前，检查最终改写草稿：

| # | 检查项 | 操作 |
|---|-------|--------|
| 1 | 意义夸大 | 替换为具体事实 |
| 2 | 含糊权威 | 标明来源或删除 |
| 3 | 含糊的深度动词 | 替换为具体动作 |
| 4 | 广告腔 | 改为客观表达 |
| 5 | 模板段落 | 保留结论，删除模板 |
| 6 | 密集 AI 黑话 | 替换为日常用语 |
| 7 | 过度使用否定对偶句 | 直接陈述 |
| 8 | 强行三段式结构 | 只保留关键点 |
| 9 | 机械同义词替换 | 使用一致术语 |
| 10 | 过度使用破折号 | 改用逗号或句号 |
| 11 | 粗体强调过多 | 删除不必要的粗体 |
| 12 | 模板化列表风格 | 更自然时合并为段落 |
| 13 | 概念堆叠标题 | 让标题更像自然对话 |
| 14 | 表情符号杂乱 | 除非用户要求，否则删除 |
| 15 | 聊天残留 | 删除 |
| 16 | 知识截止声明 | 删除 |
| 17 | 过度讨好语气 | 改为客观表达 |
| 18 | 填充短语 | 删除 |
| 19 | 过度模糊保守 | 替换为基于条件的判断 |
| 20 | 空洞结尾 | 添加真实结论或行动项 |
| 21 | 虚假的范围化表达 | 列出具体事实 |
| 22 | 叙述人称漂移 | 保持来源人称；第一人称来源仍保持第一人称 |
| 23 | 旁观者式解读 | 改写为直接经验、直接结论或实用指导 |

## 必需输出

1. 带 frontmatter 的完整改写 Markdown 文章。
2. 3-5 个备选标题。
3. 简短改写说明，列出主要结构调整。
4. 确认已应用 AI 痕迹检查清单，包括保留叙述人称、没有旁观者式解读。

## 成功标准

- 读起来像真人写的中文文章，朗读也自然。
- 没有空话、模板段落或明显 AI 腔。
- 信息密度高，每句话都有具体内容。
- 结构与翻译草稿有明显差异。
- 来源核心信息保持完整。
- 叙述人称与来源一致：第一人称来源仍保持第一人称。
